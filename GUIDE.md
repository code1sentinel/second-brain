# How to Use Your Second Brain

> A practical operating guide for this repository — based on Tiago Forte's **Building a Second Brain** and the **PARA method**.

---

## The Core Idea

Your brain is for **having** ideas, not **holding** them. This system offloads the work of remembering so your mind stays free to think, connect, and create.

Everything you capture, organise, and reference here compounds over time. The more you use it, the more valuable it becomes.

---

## The CODE Workflow

Every piece of knowledge moves through four stages:

| Stage | Question | Action |
|-------|----------|--------|
| **C**apture | Does this resonate? | Save it to `00-inbox/` |
| **O**rganise | Where does this belong? | Move it into PARA |
| **D**istill | What's the essence? | Highlight the key insight |
| **E**xpress | How can I use this? | Apply it in a project or share it |

---

## The PARA Structure

```
second-brain/
├── 00-inbox/        ← Drop zone. Everything lands here first.
├── 01-projects/     ← Active work with a deadline or outcome.
├── 03-resources/    ← Reference material you return to.
└── 04-archives/     ← Inactive stuff. Keep, don't delete.
```

### Where does this go?

Ask yourself: **"Is this tied to an active goal?"**

- **Yes, I'm actively working on it** → `01-projects/`
- **No, it's useful reference** → `03-resources/`
- **No longer relevant** → `04-archives/`
- **Not sure yet** → `00-inbox/` (decide later)

## Daily Habits

### Capture (throughout the day)
- Drop links, quotes, ideas, and notes into `00-inbox/`
- Don't organise yet — just save it
- Aim for zero friction: a single line is enough

### Weekly Review (30 min, once a week)
1. Clear `00-inbox/` — move each item to the right PARA folder
2. Review `01-projects/` — is every active project moving forward?
3. Scan `03-resources/` — did you learn anything this week to add?
4. Archive anything no longer relevant

### Monthly Review (1 hour, once a month)
1. Are your projects still your real priorities?
2. What have you learned that changed your thinking?
3. What can you archive?

## Projects vs. Resources

The most common confusion — here's the rule:

| | Project | Resource |
|-|---------|----------|
| **Has a finish line?** | Yes | No |
| **Example** | "Launch personal website by July" | "Web development notes" |
| **Lives in** | `01-projects/` | `03-resources/` |

> **Rule of thumb:** If there's no deadline or deliverable, it's a resource.

---

## Progressive Summarisation

When you re-read a note, highlight the parts that still stand out. Over multiple passes:

1. **Pass 1** — Save the raw note
2. **Pass 2** — Bold the key sentences
3. **Pass 3** — Highlight the most important bolded parts
4. **Pass 4** — Write a 2–3 line executive summary at the top

You don't have to summarise everything — only notes you return to deserve this treatment.

---

## Capturing Well

Not everything is worth saving. Apply the **Curator's Eye**: would you recommend this to a friend? If yes, save it. If no, let it go.

Good things to capture:
- Surprising facts that change how you think
- Useful frameworks, mental models, checklists
- Quotes that articulate something you've felt but couldn't say
- Resources you'll need for a future project

Bad things to capture:
- Everything (leads to overwhelm)
- Things you already know well
- Things you saved "just in case" but never revisit

---

## Expressing Your Knowledge

Your second brain is only useful if you **use** what's in it. Expression means:

- Writing a blog post or article
- Building a project, product, or tool
- Sharing knowledge in a conversation or presentation
- Making a decision informed by your notes

> The goal is not a perfect archive — it's a system that helps you **do** things.

## Principles to Remember

1. **Capture first, organise later.** Don't let the perfect system stop you from saving something now.
2. **Organise by actionability, not topic.** PARA is about *what you're doing*, not *what a thing is*.
3. **Return on attention.** Only invest time distilling notes you actually use.
4. **Intermediate packets.** Every note, summary, or outline is a reusable building block for future work.
5. **Done is better than perfect.** A rough note in the right folder beats a perfect note that never gets written.
